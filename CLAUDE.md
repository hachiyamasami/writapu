# Claude Project Guidelines

## Communication Style
- **Language**: すべての対話、説明、提案は**日本語**で行ってください。
- 専門用語を除き、基本的には日本語で簡潔に回答してください。

## Git & Issue Management Rules
- **CRITICAL**: ユーザーの明示的な許可なく `git push` や `gh pr create` を実行しないでください。
- **MANDATORY**: すべてのコミットメッセージには必ず GitHub Issue 番号を含めてください（例: `fix #123: ログインバグの修正`）。
- 作業開始前に `gh issue list` を確認し、関連する Issue を特定してください。
- 修正完了後、コミットまで行い、「Issue #[番号] でコミットしました。GitHubへプッシュしますか？」と日本語で確認してください。

[出力に関する厳守事項]

既存ファイルの修正を依頼された際、特に指示がない限り、ファイル全文を再出力してはならない。

ファイル全文を出力するとトークン制限に触れるため、修正が必要な『該当するMarkdownのブロック』のみを出力し、前後は省略すること。

「全文が必要ですか？」という確認も不要。常に最小限の差分のみを提示せよ。
